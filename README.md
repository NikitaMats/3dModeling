# TestCase_Sputnik
---------------------------------------------------------------------
							 0. Предисловие							 
---------------------------------------------------------------------
Данный проект является тестовым задание для компании «Спутник». 
Решения принятые в процессе реализации програмного комплекса(ПК), 
берут во внимание необходимость продемонстрировать навыки соискателя,
и являются излишними для такого простого ПК.

---------------------------------------------------------------------
						1. Архитектурные принципы					
---------------------------------------------------------------------
Согласно техническому заданию(ТЗ), при разработке ПК был использован,
паттерн MVVM (Model-View-ViewModel). Который используется для 
разграничения логики и элементов интерфейса.
Так же в процессе разработки были приняты во внимание принципы SOLID.
Которые сделали код более удобным для даленейшей работы и 
маштабирования. Как часто говорил мой преподаватель. Код пишется 20%
времени, а остальное время читается.

---------------------------------------------------------------------
						   2. Технологии WPF						 
---------------------------------------------------------------------
Для зарзаботки UI из предложенных в ТЗ вариантов было выбран WPF,
ввиду наличия у меня опыта работы с ним и его предпочтительностью 
перед WinForms. Для графики был использован встроенный интрумент 
Viewport3D.

---------------------------------------------------------------------
						   3. Технологии GIT						 
---------------------------------------------------------------------
Для удобной работы так же была использованная система контроля версий
GIT. Историю коммитов можно посмотреть в данном репозитории. Система
ветвей не была использованна так как является излишней при разработке
одним человеком и не требуется по ТЗ.

---------------------------------------------------------------------
						   4. Сборка проекта							
---------------------------------------------------------------------
В проекте нет мест требующих вмешательства в код перед запуском на
новом устройстве поэтому дополнительные действия не требуются.
Запуск возможен сразу после компиляции проекта в VS, так как создан
файл gitignore который поумолчанию не клонирует папку ./bin.
